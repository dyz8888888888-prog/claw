核心结论 -- 龙头的间接门控角色

发现 1
龙头不直接参与评分 -- 没有"龙头加分"的线性权重系数。龙头的角色是通过共识阶段间接影响，而非作为独立维度加/减分。
发现 2
阶段3(封板)是关键阈值 -- 龙头涨停+有跟班 -> 阶段升至3以上 -> 主线确认 -> 策略1外溢策略打开。龙头涨停但无跟班(孤狼)只能到阶段2，无法触发最强策略。
发现 3
策略2(板块扩散)对主线要求宽松 -- 阶段1(酝酿)的主线观察即可，但会被概念质量过滤(扩散<30%或RRG落后 -> 不追)。
发现 4
信号引擎升一级 -- 主线内触发B级信号自动升A，分数x1.2。这是龙头影响力"下沉"到信号层的入口。
发现 5
Dashboard龙头排行是独立展示层 -- /api/dragons 基于共识数据独立计算热度分，用于人工视觉确认，不参与自动化决策。
发现 6
策略3(错杀修复)完全不依赖龙头 -- 退潮/冰点市场的超跌修复策略只关心转债跌幅和正股抗跌，不检查主线。

完整决策链路 -- 以"龙头涨停"为例

龙
某概念龙一涨停 +10%，龙二 +3%，龙三 +1.5%
-> limit_up_count=1, follower_count=2, dragon_sc_pct=10.0
+
阶
共识阶段 = 4 (扩散)
-> _compute_stage: limit_up_count=1 AND follower>=2 -> stage=4
+
主
主线确认 (MAINLINE_CONFIRMED)
-> set_mainlines: stage>=3 -> _mainline_concepts.add(concept)
+
策
该概念下所有满足条件的转债 -> 触发"埋伏"信号
-> 策略1(外溢): mainline=CONFIRMED + stock=UNBUYABLE + risk=OK -> 埋伏
-> 策略2(扩散): mainline=CONFIRMED + stock>=STRONG + risk=OK -> 埋伏
-> 信号引擎: B级信号自动升级为A级

三大策略的门控条件对比

策略1 -- 正股涨停外溢
主线要求: 必须 主线确认/观察

正股: 涨停(>=9.5%)
风险: 可做
市场: 非冰点

mainline >= WATCH
stock = UNBUYABLE
risk = OK
market != freeze
策略2 -- 板块扩散
主线要求: 主线观察即可

正股: 强势(>=3%)
风险: 可做
概念质量: 扩散>=30%

mainline >= WATCH
stock >= STRONG
risk = OK
diffusion >= 30%
策略3 -- 错杀修复
主线要求: 不需要

转债: 跌>=2%
正股: 抗跌(>-3%)
成交额: >=0.2亿

cb_pct < -2%
stock_pct > -3%
amount > 0.2亿
无主线依赖

影响力强度总结

决策环节 龙头参与方式 影响力 说明
策略1(外溢) 门控 主线确认 = 必要条件 决定性 - 能/否触发 没有主线确认 -> 策略1完全不触发
策略2(扩散) 门控 主线观察 = 必要条件 决定性 - 能/否触发 无主线观察 -> 策略2在启动市不可用
概念质量过滤 扩散指标(跟班率) + RRG 附加门槛 扩散<30% 或 RRG落后 -> 过滤(不追)
信号引擎升级 主线内 B->A 等级提升 主线内的滞涨信号优先级提升
策略3(错杀) 判定 不参与 无影响 错杀修复不检查主线
Dashboard 排行 独立计算热度分 仅展示 /api/dragons 用于人工确认
