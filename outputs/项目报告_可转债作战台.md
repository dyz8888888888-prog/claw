# 可转债作战台 — 项目描述报告

> 生成时间：2026-06-27 | 版本：V8.1

---

## 一、项目定位

**可转债作战台**是一套面向A股可转债日内交易的实时监控与决策辅助系统。围绕"选债→盯盘→情绪感知→信号生成→复盘"的完整交易闭环，整合通达信行情、东方财富转债数据、Fuyao涨停池、开盘啦情绪数据四个异构数据源，提供从盘中实时信号到盘后全量复盘的端到端能力。

**核心目标**：在可转债T+0交易场景下，降低信息过载，提高信号命中率，辅助交易决策。

## 四、核心模块详解

### 4.1 选债与数据融合

| 模块 | 文件 | 功能 |
|------|------|------|
| BondSelector | `core/bond_selector.py` | 从全市场500+可转债中筛选监控池（规模<10亿/溢价率-5%~35%） |
| DataFusion | `core/data_fusion.py` | 多源数据合并：TDX行情 + 东方财富转股价/溢价率 |
| RollingWindow | `scheduler/rolling_window.py` | 60轮滚动窗口，提供成交量基线/价格趋势 |

### 4.2 信号引擎

| 信号等级 | 含义 | 触发条件 |
|----------|------|----------|
| **S级** | 极强 | 正股涨停+转债联动、板块共振突破 |
| **A级** | 强 | 连续放量拉升、溢价率突变、主线概念共振 |
| **B级** | 中 | 缩量急跌反弹、股债联动偏差 |
| **C级** | 弱 | 尾盘洗盘、超跌反弹 |
| **D级** | 观察 | 量价异动、概念孤狼（不推送，仅记录） |

信号经 `AlertManager` 去重过滤（同债10分钟内只保留最强信号），B级日上限20只。

### 4.3 情绪引擎（六维）

参考超短复盘框架，整合盘中TDX快照 + 盘后KPL全量数据：

| 维度 | 指标 | 盘中源 | 盘后源 |
|------|------|--------|--------|
| 涨停强度 | 涨停数/跌停数/首板连板分布 | TDX转债池 | KPL全量 |
| 打板质量 | 炸板率/封板成功率 | 盘中估算 | KPL精确 |
| 接力情绪 | 连板率/晋级趋势 | — | KPL |
| 亏钱效应 | 大幅回撤/天地板/昨日破板 | — | KPL |
| 赚钱效应 | 昨日涨停今日表现/百日新高 | — | KPL |
| 市场广度 | 涨跌比/成交额 | TDX快照 | KPL补充 |

**情绪阶段映射**（唯一市场状态源）：
```
过热 → climax   | 活跃 → ferment  | 温和 → startup
退潮 → retreat  | 冰点 → freeze
```

### 4.4 决策管道（4阶段）

| 阶段 | 功能 | 产出 |
|------|------|------|
| 市场状态判定 | 情绪 → 5阶段 | 仓位建议 |
| 转债信号扫描 | 80只候选逐只判定 | 埋伏清单/卖出清单 |
| 概念传导分析 | 概念共识追踪 | 传导链路/龙头板块 |
| 综合建议 | 多维度融合 | 操作建议 + 禁入清单 |

输出三类决策：
- **埋伏 (Ambush)**：低吸待涨机会
- **卖出 (Sell)**：止盈/止损信号
- **禁入 (Forbidden)**：高风险回避（妖债/强赎/溢价异常）

### 4.5 龙头排序（三维）

```
DragonRanker: 封板时间(35%) + 封单金额(35%) + 连板数(30%)
→ 龙一/龙二/龙三 排行
→ 联动转债推荐
```

### 4.6 共识追踪

实时追踪概念板块的热度传导链，从 Fuyao 涨停池聚合板块数据（≥2只涨停门槛），模糊匹配标准化板块名称，输出概念强度排行和传导路径。

## 五、数据源矩阵

| 数据源 | 用途 | 获取方式 | 更新频率 |
|--------|------|----------|----------|
| **通达信 TDX** | 转债快照/正股涨跌/规模 | V7.73 PYPlugins (tq API) | 3秒/轮 |
| **东方财富** | 转股价/溢价率/强赎 | mx-finance-data | 按需 |
| **Fuyao MCP** | 涨停池/炸板池/板块指数 | REST API (90s节流) | 盘中实时 |
| **开盘啦 KPL** | 涨停原因/龙头/情绪/封单 | kaipanla-crawler | 盘后日级 |

**盘中/盘后混合策略**：
- 09:30-15:00 → TDX快照 + Fuyao涨停池（轻量实时）
- 15:30后 → 自动切换KPL全量数据（炸板率/晋级率/封单完整）

## 七、部署方案

### 方案一：CloudStudio 永久快照
- **链接**：每次部署生成独立 sandbox URL
- **机制**：`gen_live_html.py` 从本地 Flask API 拉取全量数据 → 内嵌为自包含 HTML → 部署至 CloudStudio
- **刷新**：自动化每5分钟执行（仅交易日）
- **优点**：永久在线，无需电脑开机

### 方案二：localtunnel 实时隧道
- **链接**：`https://dyz-cb-battle.loca.lt`
- **机制**：Flask 5000 端口通过 localtunnel 暴露公网
- **限制**：需电脑保持运行 + TDX 在线
- **优点**：实时数据，3秒刷新

## 八、日报子系统（可转债日报）

独立运行的盘后分析管道：

```
市场快照(09:40/10:00/10:32/11:40/13:30/14:15) 
→ 行情分类(A~F 6种) 
→ 候选评分(振幅25%+规模20%+流动性20%+溢价15%+方向20%) 
→ 概念加分(Σ概念热度/√n) 
→ 日报生成
```

**行情分类**（三级递进识别）：
| 类型 | 描述 | 09:40 | 10:00 | 10:32 |
|------|------|-------|-------|-------|
| A | 强势普涨 | 粗判50% | 确认80% | 定型90% |
| B | 温和偏强 | ↓ | ↓ | ↓ |
| C | 微涨分化 | ↓ | ↓ | ↓ |
| D | 横盘震荡 | ↓ | ↓ | ↓ |
| E | 微跌分化 | ↓ | ↓ | ↓ |
| F | 弱势普跌 | ↓ | ↓ | ↓ |

**评分算法**（V7 波动优先）：
- 振幅25% + 规模20% + 流动性20% + 溢价15% + 方向20%
- 加分项：微盘+15 / 小盘+8 / 活跃+15 / 超低溢价+10

## 十一、关键设计决策

1. **盘中轻量、盘后全量**：交易时段用TDX快照+Fuyuo（低延迟），盘后切换KPL全量（炸板率/晋级率精确）
2. **情绪引擎为市场状态唯一源**：不再依赖独立的 MarketStateClassifier，情绪 → 5阶段映射 → 决策管道
3. **波动优先评分**：V7核心理念——有波动才有收益，溢价是约束不是目标
4. **妖债单独处理**：溢价>100%或价格>800元标记为妖债预警，不参与正常排名评分
5. **429限流保护**：Fuyao 3层并发改节流 (60s→90s/120s)，HTTP 429指数退避重试
6. **交易日历过滤**：自动化仅在A股交易日执行，避免周末/节假日空跑
