## 一、项目目标

每天开盘后 6 个快照时间点，自动生成一份 **可转债 x 涨停概念日报**，包含：

1. **行情类型判定** — 6种 A-F 类型，决定概念 / 基本面的策略权重
2. **TOP10 日报排名** — 综合溢价率、剩余规模、流动性、振幅、概念热度
3. **妖债预警** — 溢价>100% 或价格>800 的标的单独列出
4. **对比验证** — 日报 TOP10 vs 实际涨幅 TOP10 命中率

## 六、评分算法 (完整公式)

### 6.1 候选分 (scorer.py — 唯一真相源)

```
候选分 = 溢价分(35%) + 规模分(30%) + 流动性分(20%) + 振幅分(15%) + 加分项

各因子 min-max 归一化到 [0, 100]:
  - 溢价率: 越低越好 (reverse)
  - 剩余规模: 越小越好 (reverse)
  - 成交额: log10后越大越好
  - 振幅: 越大越好

加分项 (累积):
  - 微盘 bonus: scale < 2亿 → +15
  - 小盘 bonus: scale < 5亿 → +8
  - 超低溢价 bonus: premium < 5% → +25
  - 低溢价 bonus: premium < 10% → +15
```

### 6.2 概念分

```
概念分 = min(80, max(20, Σ热度分 / √概念数))

概念热度分: 从 i问财 "今日涨幅" 股票的所属概念聚合
  每个概念的热度 = Σ该概念下个股当日涨幅
  归一化到 [0, 100]

无概念数据时: 回退基准 40 分
```

### 6.3 最终分

```
最终分 = 候选分 × 候选权重 + 概念分 × 概念权重
```

D型行情示例: 最终分 = 候选分 × 0.70 + 概念分 × 0.30

### 6.4 妖债判定

```
满足任一即标记为妖债 (不参与正常排名):
  1. 溢价率 > 100%
  2. 价格 > 800 元
```

## 九、运行方式

### 9.1 完整流水线 (一条命令)

```bash
cd 可转债日报
python scripts/generate_daily_report.py
```

自动完成:
1. 加载 `cb_top50_full.json` (行情 + 规模)
2. 调用 i问财获取实时行情快照 (指数/涨跌比/涨停数)
3. 加载概念映射 + 概念热度缓存
4. 妖债过滤 → 候选评分 → 行情分类 → 概念分 → 最终排名
5. 输出日报 + 对比报告到 `日报对比_*.md`

### 9.2 数据刷新

```bash

python scripts/scale_fetcher.py

# 刷新概念映射 (每周)

```
